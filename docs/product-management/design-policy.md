# UI/UX デザインポリシー

このドキュメントは、SlideCraft プロジェクトにおける UI/UX デザインの基本方針を定義します。

## デザイン哲学

「見た目がきれい」よりも「使いやすい」を優先します。どれだけ視覚的に洗練されていても、操作性に問題があれば「なんか使いづらい」という印象を与えてしまいます。

優れたユーザー体験は、以下の3つの原則によって実現されます。

1. 操作がブロックされない
2. 操作に対してすぐにフィードバックがある
3. すぐに元の状態に戻れる

## 操作がブロックされない

ユーザーの操作を必要以上に制限しないことで、ストレスのない体験を提供します。

何かの処理中であっても、他の操作は可能な限り許可します。処理に時間がかかる場合は、バックグラウンドで実行し、ユーザーは別の作業を続けられるようにします。

時間のかかる操作には、キャンセルボタンを提供します。ユーザーが「やっぱりやめた」と思った時に、簡単に中断できるようにします。

モーダルダイアログは操作を完全にブロックするため、多用しません。使用する場合でも、すぐに閉じられるように ESC キーや背景クリックでの閉じる動作を必ず実装します。情報の表示だけならポップオーバーやトーストなど、より軽量な UI パターンを検討します。

## 操作に対してすぐにフィードバックがある

ユーザーの操作が正しく受け付けられたことを即座に示すことで、安心感を与えます。

ボタンのクリック、フォームの入力など、あらゆる操作に対して視覚的なフィードバックを提供します。「これ押せてる?」という疑問を抱かせないことが重要です。

ローディング中、成功、エラーといった状態を明確に表示します。処理に時間がかかる場合は、進捗状況を示します。

表示速度も大事です。レイアウトシフトを最小限に抑えつつ、表示できる部分から順次表示します。Skeleton UI の活用や、重要なコンテンツの優先的な読み込みを検討します。

ユーザーの操作が成功すると仮定し、即座に UI を更新する楽観的 UI (Optimistic UI) の活用も有効です。サーバーからのレスポンスを待たずに反応することで、体感速度を向上させます。ただし、失敗時のロールバック処理も必ず実装します。

## すぐに元の状態に戻れる

ユーザーが安心して操作を試せるよう、元の状態への復帰を容易にします。

操作後に前の画面に戻るための導線を必ず用意します。戻るボタンは視認しやすい位置に配置します。

リスト画面から詳細画面に遷移し、戻った時は同じスクロール位置に戻るようにします。フォームの入力内容は、他の画面に行って戻ってきても保持されているべきです。Twitter/X がポストの詳細を開いてから戻ってもタイムラインの同じ位置に戻れるのが良い例です。

元に戻るコストを考慮して適切な UI パターンを選択します。コストの高い順に、ページ遷移、モーダル、ポップオーバーとなります。一時的な情報表示にはページ遷移ではなく、モーダルやポップオーバーを使います。

重要な操作、特に削除などには、確認ダイアログまたは取り消し機能を提供します。即座に削除するのではなく、「〇〇を削除しました」というトーストに「元に戻す」ボタンを表示する方が、ユーザーフレンドリーです。

## 操作の予測可能性

ユーザーが操作する前に、何が起こるかを予測できることも優れた UI の条件です。

同じ操作には同じ結果を返します。アプリケーション全体で UI パターンを統一します。

ボタンのラベルは具体的な動作を示します。「OK」ではなく「保存」「削除」「キャンセル」など、何が起こるかが明確な言葉を使います。

リンクは下線やカラーで示し、ボタンは押せることが分かるデザインにします。無効化された要素は明確に区別できるようにします。

データの削除など、元に戻せない操作には確認を求めます。ただし、確認ダイアログの多用は避けます。可能であれば、Undo 機能や論理削除で対応します。

## 実装時のチェックリスト

新しい機能や画面を実装する際は、以下の点を確認してください。

- 処理中でも、他の操作は可能か?
- 時間のかかる操作はキャンセルできるか?
- モーダルは本当に必要か? ポップオーバーで代替できないか?
- ボタンのクリックやフォームの入力に即座に反応しているか?
- ローディング状態は明確に示されているか?
- 前の画面に戻る導線は明確か?
- スクロール位置や入力内容は適切に保持されているか?
- ボタンのラベルは具体的で、何が起こるかが予測できるか?
- 破壊的な操作には適切な警告または Undo 機能があるか?

## まとめ

このデザインポリシーは、「見た目の美しさ」よりも「使いやすさ」を優先する姿勢を明確にするものです。

ユーザーの操作を邪魔せず、操作に対して即座に応答し、いつでも安心して元の状態に戻れ、何が起こるかが予測できる。これらの原則を守ることで、ユーザーにとって本当に使いやすいアプリケーションを提供できます。
